The three rules of clean code

1: Everyone else who is looking at your code is an idiot

  1. "This function is really clever if you understand it!"

    The idiots won't. Make it readible

  2. "It may be less readible but it saves clock cycles"

    You better have a really good reason for blindsiding the idiots like that

  3. "I'm not going to move this into a function just because some people don't get regex"

    Most really smart people don't understand regex. Stop being a dick.

2: Code is read far more than it is written

  1. "No it's not"

    If your code has never had a bug ever then ok, you're right

  2. "I'm not refactoring my working code, it might break"

    If refactoring breaks your code, your code is already broken

3: Use standard or jslint

  1. "We use JsHint"

    Stop then

Size matters

  1. functions - make them small

  3. modules - make them small

Naming

This isn't a crossword, write in plain english

  1. Names can be long

  2. If the name is too long, the function is doing too much

  3. "If there's a shorter, but yet descriptive way to name the function, then the function name is too long." - some dude on stack overflow
